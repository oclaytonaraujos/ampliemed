Objetivo obrigatório

O sistema deve funcionar exclusivamente com Supabase como backend e como única fonte de verdade dos dados.

Não deve existir dependência de armazenamento local para persistência de dados, incluindo:

localStorage

sessionStorage

IndexedDB

mocks persistentes

arquivos locais usados como base principal

qualquer outro banco local

Todo dado do sistema deve ser persistido, consultado, atualizado, removido e sincronizado somente pelo Supabase.

Segurança obrigatória

Inclua no SQL toda a configuração de segurança necessária para produção real.

Deve incluir:

Não use policies genéricas ou superficiais.
Crie regras coerentes com o funcionamento real do sistema.
